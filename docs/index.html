Quickstart

From "never heard of Hugin" to "PR submitted" in roughly five minutes, assuming you have a Modbus device on your network you want to write a driver for.

Prerequisites

  • A device that speaks Modbus TCP, MQTT, or HTTP. Most modern inverters, batteries, EV chargers, and smart meters qualify.
  • The device on the same LAN as the machine where you'll run the agent. (Or somewhere reachable from it.)
  • A GitHub account. The workbench signs in with GitHub so PRs land in your name.
  • One of: macOS with Homebrew, Windows with Scoop, or Linux + Go 1.25+.

1. Sign in to the workbench

Open hugin.sourceful-labs.net/workbench.html and sign in with GitHub. The OAuth scope is public_repo only — used to fork srcfl/hugin-drivers and open a PR in your name. Sourceful never reads your private repos.

2. Install the agent

The agent is a 12 MB Go binary. It listens on localhost:19090 and gives the browser real LAN access — Modbus, ARP, HTTP probes, Lua sandbox. Pick one install path:

macOS / Linux — Homebrew

brew install srcfl/tap/hugin-agent
hugin-agent

Windows — Scoop

scoop bucket add srcfl https://github.com/srcfl/scoop-bucket
scoop install hugin-agent
hugin-agent

From source

go install github.com/srcfl/hugin-agent/cmd/hugin-agent@latest
hugin-agent

First run prints a pairing URL and auto-opens your browser to /settings.html#agent_url=…&token=…. Click Save pairing; the workbench takes it from there.

Pairing without copy-paste. The agent's deep-link fragment carries the URL + token. Settings.html ingests them directly and never sends them to a server — your token stays in your browser's localStorage.

3. Pick a device

On the workbench's empty state, click Scan my LAN. The agent sweeps your subnet via ARP + Modbus FC43 + HTTP banner-grab and shows everything it found. Pick the row that matches the device you're targeting.

If your device isn't on this LAN (or the scan can't reach it), use Paste device specs / manual instead — drop in a register list or a vendor manual snippet and the AI starts from there.

4. Probe a register

Click Probe register. Type an address (decimal or 0x4000), pick a decode kind (i32_be, u16, etc.), and hit Probe. The agent runs a single Modbus FC03 read and shows you the raw bytes + decoded value. Probed values are ground truth — they override anything the AI assumes.

5. Let AI draft the driver

Click AI: Generate from probed. Claude reads the device info + probed register values + the authoring guide's sign conventions and outputs a complete Lua driver in the editor. Inspect it, tweak it.

6. Test against the device

Adjust the duration slider (5–120s) and hit ▶ Run. The agent loads your Lua into a sandbox and runs driver_init followed by driver_poll in a loop. Emissions stream into the Live panel as they fire — no waiting for buffered output if you're paired in remote (NATS) mode.

7. Submit as PR

When the Live panel looks right, click Submit as PR. The workbench forks srcfl/hugin-drivers to your account (if not already), commits the Lua file, and opens a PR with your name on the commit. Maintainers review and merge.
